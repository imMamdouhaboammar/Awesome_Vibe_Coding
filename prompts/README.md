# Prompts

A structured library of tested prompts for various domains, optimized for clarity, constraints, and deterministic outputs.

## Categories

- [Coding Prompts](./coding/) — Code generation, refactoring, test writing, and bug hunting.
- [Design Prompts](./design/) — UI layouts, styling, color palette generation, and component structures.
- [Writing Prompts](./writing/) — Editing, documentation drafting, tone adjustment, and structural outlining.
- [Research Prompts](./research/) — Codebase explanation, paper summarization, and tech-stack comparisons.
- [Automation Prompts](./automation/) — Scripting, CI/CD configuration, and build-run pipeline generation.

## How to Add a Prompt

To add a prompt:
1. Create a markdown file in the appropriate category subfolder (e.g., `prompts/coding/refactor.md`).
2. Copy the template structure from [`/templates/prompt-template.md`](../../templates/prompt-template.md).
3. Fill in details and register it in the subfolder README.
